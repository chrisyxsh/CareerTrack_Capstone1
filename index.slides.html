Castone Project I - Geo-Mapping Yelp Businesses & Predict Rank Stars¶

Project Goals¶

When someone wants to move to some place to live, or travel to some city, or have a long vacation to explore more places, they usually have some questions such as "Where to go?", "How's the place?","What's funny in this area?", etc.. People can use yelp, google, or tourist guides to find some restaurants, sceneries.

But the information they searched most probably is about a specific spot. How about an area, for example, within 25 square miles? This area has many museums and souvenir stores, that place has many Cafe and restaurants, another place has many gyms, hotels, and rental apartments. If people can make a sketch or even a portrait for places, they can make better plans for saving money and saving time.

This project will statistically extract business information, using the data set offered by the Yelp company, covering 7 USA states and 354 cities, and 127189 businesses which are grouped into nine business category groups such as catering, fitness, shopping, etc., showing all statistical information on a three layers dynamic map.

This project also will prove that there is a strong positive relationship between the number of reviews and the number of review votes.

At last, we will predict businesses stars ranked by users from their reviews.

Data Source¶

The data set comes from the Yelp's Open Dataset. The documentation of the data set is here.

The Yelp dataset offered is a subset of businesses, reviews, and user data for use in personal, educational, and academic purposes. There are 5,200,000 reviews, 174,000 businesses, and 200,000 pictures, covering 11 metropolitan areas.

Data Cleaning¶

The Yelp data set contains five JSON files which are "business.json", "review.json", "checkin.json", "tip.json", "user.json" and "photos.json". This project will focus on the data on businesses and reviews.

Tidy data on seven states in USA¶

There 174567 observations in the business dataframe, each observation represents a business entity such as a restaurant or a fitness club. Those businesses are located in 68 states or provinces in the USA and other countries. Here we just need states of USA. In addition, the number of businesses should be big enough to portray a place. So we choose states in the USA which has more than 1000 businesses in our data set. Then we will have a new data set containing 127189 businesses in seven USA states whose abbreviations are: 'AZ','IL','NC','NV','OH','PA' and 'WI'.

Businesses grouped into nine category groups¶

In the data set, the 'categories' variable of a business is defined with a random way. Each business has its own content of the 'categories'. For a good overview on all businesses, we separate all businesses into 9 category groups which are: 'Entertainment & Catering', 'Nearby Services','House-inside Services','Fitness, Health Care', 'Shopping', 'Education', 'Auto, Transportation', 'Public Facilities', 'Other Professional Service'. 'Nearby Services' includes the routing services people need such as hair saloon, pet care, bank, post, etc., 'House-inside Services' include all serviced people need for their house such as plumbing, cleaning, repairing etc.. We combine fitness and medical health care together, and combine car services with transportation facilities such as airport, bus station.

Grouping businesses is a quite complicated task, and can be grouped by different view angle. It's important to describe a place. Actually, there is much more job to do on this issue far beyond what we have done in this project.

ZIP code and City name corrected via the Google Geocoding API¶

There are lots of missing values or error values in the contents of variables, like 'address', 'postal_code', 'city', 'neighborhood'. The 'address' has 5974 missing values, the 'neighborhood' has 89680 missing values, and the 'postal_code' has 410 missing values.

Considering the Google Geocoding API Usage Limits, we choose to fill the missing values of the 'postal_code'.

We also need to get the right values of the 'city', '110 Las Vegas' definitely is not a right city name. If we want to do statistics on the state level, and the city level, we need right cities' names.

We cannot correct all cities' names with iterating each observation. After filling the missing values of the 'postal_code', we can group observations by 'state', 'postal_code' and 'city', and get one set of 'latitude' and 'longitude', representing the location coordinates of each city, to inquire the city name from Google Geocoding API. Thus the number of requests will below the 2500 limitation.

Add on review-votes correlated with reviews in the business dataframe¶

There are near 5,200,000 reviews in the 'reviews.json'. It's too large for our project. We extract variables related to review votes, which are 'useful', 'funny', 'cool', and merge them with the business dataframe. Thus we have a cleaned dataframe containing major businesses' location information like 'state', 'city', 'postal_code', 'latitude' and 'longitude'. We also have some social measurement information for each business like 'review_count' and three kinds of review votes('useful', 'cool' and 'funny').

Split 5 million reviews' text into a small data set for predicting ranking stars¶

Five million observations for classification in this project is too big. So we split it three times using Scikit-learn's 'train-test-split', and finally get a small data set with reasonable size of near 100K observations. We will split the data set again to create our training and test data sets, and use them to predict Yelp business ranking stars using Multinomial Naive Bayes methods and Logistic Regression methods.

Hypothesis Test¶

Null Hypothesis¶

The number of reivews and the number of review votes are completely uncorrelated.

Pearson Correlation Coefficient Statistic and Bootstrap method¶

We will use the Pearson Correlation Coefficient as the statistic to do the hypothesis test.

At First we will calculate the Pearson Correlation Coefficient with current samples in the business dataframe.

In [93]:
def pearson_cc(x,y):
    return np.corrcoef(x,y)[0,1]

def draw_bs_relation(x, y, func,args=None,size=1,corr=False) :
    bs_reps = []
    inds = np.arange(len(x))
    
    for i in range(size):
        if corr :
            bs_inds = np.random.choice(inds,size=len(inds))
            bs_x, bs_y = x[bs_inds], y[bs_inds]
        else :
            bs_x = np.random.choice(x, size=len(x))
            bs_y = np.random.choice(y, size=len(y))
        
        if args:
            bs_reps.append(func(bs_x,bs_y,*args))
        else:
            bs_reps.append(func(bs_x,bs_y))

    return bs_reps

Tuning hyperparameters of models¶

The 'min_df' parameter of CountVectorizer¶

With the performance of the initial model, we can firstly tuning the cutoff parameter of CountVectorizer, which is 'min_df'.

We construct a cumulative distribution of document frequencies (df). The $x$-axis is a document count $x_i$ and the $y$-axis is the number of words that appear less than $x_i$ times. For example, at $x=5$, the point representing the number of words that appear in 5 or fewer documents. Look for the point at which the curve begins climbing steeply. This may be a good value for min_df.

The 'ngram_range' parameter of CountVectorizer and the 'alpha' parameter of MultinomialNB¶

Next, we will use GridSearchCV function to tune the ngram_range parameter of CountVectorizer, and the alpha parameter of MultinomialNB together.

Considering the big computation cost of GridSearchCV, we will split 'reviews' and 'stars' into a dataset with near 10000 observations.

We will use a pipeline, a parameter grid as the input of GridSearchCV, and define the value of cv parameter equals 5 which means GridSearchCV will implement 5-fold cross-validation.

The pipeline includes two steps which are CountVectorizer and MultinomialNB. The parameter grid is a dictionary with parameter names as keys and lists of parameter settings to try as values, those parameters correspond the two models in the pipeline.

We will use the fit method of GridSearchCV with the dataset splitted previously, and then get the best model score and best parameters via the attributes of GridSearchCV, which are best_score_ and best_params_.

Using KFold and cross_val_score with a cheaper computation cost¶

The computation cost of GridSearchCV is quite expensive. We can use KFold function and cross_val_score function to tune parameters of a model. This time we will tune the alpha for MultinomialNB, and we will use F1_score as the metric measuring the performance of the model with different alpha values.

In [22]:
alphas = [.1, 1, 5, 10, 50]
best_alpha = None
maxscore=-np.inf
seed = 2014

for alpha in alphas:
    kfold = KFold(n_splits=5, random_state=seed)
    model = MultinomialNB(alpha=alpha)
    scoring = 'f1_weighted'
    #scoring = 'accuracy'
    #scoring = 'neg_log_loss'
    
    results = cross_val_score(model, X_train, y_train, cv=kfold, scoring=scoring)
    score = results.mean()
    print("alpha: {}".format(alpha))
    print("{} score: {}".format(scoring,score))
    if score > maxscore:
        maxscore = score
        best_alpha = alpha

Discussion and further work¶

Mapping Data as Dynamic Application¶

In this project, we dynamically do Geo-mapping 127,189 businesses in Yelp, grouped by their business categories, displaying business information for a whole state, or for a city, or for an individual business.

There is a limitation that we can run this map exchanging data with pandas in a dynamic Jupyter notebook. But we cannot do this in a static report with html or pdf format.

This is a trying for applying data science to dynamic application. Data science as an app, or as a platform, may be a good front-end service for massive normal users, not just a tool running in back-end used by organization or companies. Obviously, there is a lot of work to implement it. But, why not?

Text Analysis Classification¶

Cost-effective VS Higher-performance¶

The LogisticRegression may be a better choice than MultinomialNB with rich and high-performance computing resources. On the other hand, the MultinomialNB is good for quick analysis and is much more cost-effective.

Preprocessing X features¶

We tuned min_df and ngram_range of CountVectorizer on extracting X features. In fact, stop_words=english of CountVectorizer is used in this project without getting any improvement on model performance. We also used Scikit-learn's TfidfVectorizer replacing CountVectorizer, and got worse performance.

We can do more on preprocessing, but now, all of the preprocessing jobs we have done are based on words distribution, we may consider if there are better practical methods of preprocessing, which can help extracting more specific documents' meaning. Meaningful extraction can be a good foundation to improve recommendation system.
